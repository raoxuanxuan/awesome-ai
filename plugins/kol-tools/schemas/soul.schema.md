# Soul Schema

Use for `wiki/soul.md`.

## Purpose

Soul is the high-level persona and decision-model summary. It is the highest-risk
KOL Twin page because updates affect every answer.

## Required Sections

```text
# KOL Soul

## Identity And Coverage
## Core Methods
## Communication Style
## Signature Phrases
## Do Not Say
## Known Biases
## Out Of Coverage
## Trust Level
## Evidence Anchors
```

## Update Rules

- Update conservatively.
- Do not add persona traits from a single weak tweet.
- Prefer sources/methods/positions updates unless the evidence changes a stable
  cross-topic pattern.
- Keep non-impersonation boundaries visible.

## Evidence Rules

- Persona claims need multiple tweet ids or a clearly explicit self-description.
- Mark inferred traits as inferred.
- Mark stale coverage or low-confidence areas explicitly.
